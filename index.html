Project I

This is Devops in AWS using CodeCommit as source code repository. Generated SSH private and public key and uploaded public key to AWS account to serve as user name and private key as password to access Code Commit. The code is pushed to Code Commit and CodeBuild test and validate the code. The code that fails the test will be roledback, debugged and pushed back to Code Commit. And CodeDeploy to deploy the application to Ec2 Instances. CodePipeline to cordinate the three stages from source to destination taking artifactory input from S3 bucket.

Project II

This project is devops focused. It uses Elastic Container Service with other developer tools in AWS to automate application deployment. The data is stored in DynamoDB global table in two region with stream enabled. Elastic Container Registry contain docker images. CodeCommit host source code repository. CodeBuild used buildspec.yml to build the application and CodeDeploy deploy the app to Elastic Container Service in two regions. CloudWatch monitor resources utilization. Route 53 route traffic to two Elastic load balance in two regions and the Elastic load balance is configured with ssl certification.

Project III

This project involves management of container(pods) orchestration using Elastic Kubernetes Service on AWS. eksctl command tool is used to create eks cluster and nodegroup with specific AMI. The nodes are autoscaled with scaling policy to scale out or scale in as needed. Eksctl utility was used to create kube config file to enable kubectl communicate with API control plane securely. Calico network service was applied using calico manifest. Ingress controller created to expose the service mapping the pods through Network load balance. Iam identity mapping with roles waws created within eks cluster for users. Prometheus was deployed within the cluster to gather application specific metrics and CloudWatch was used to monitor every other component of EKS eco system.

Project IV

This Project is on using of Real User monitoring and Synthetic monitoring tool to monitor a java package web container application running on Tomcat server embedded with in memory SQl database. The real user monitoring tool focus on monitoring transactions throughput, Response time and error rate using Riverbed software of SteelCentral SaaS to collect the data metric and push to Datagod for dashboarding. And the synthetic monitoring uses Pingdom software to monitor uptime of the web application .
